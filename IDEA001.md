# 笔记：Contextual MWU-RL —— 面向对抗性金融市场的联合量化架构

## 1. 理论发轫：从公式的同构性出发

在构建复杂的决策系统时，我们通常会在两个看似平行的领域中寻找答案：**在线学习 (Online Learning)** 与 **强化学习 (Reinforcement Learning, RL)**。通过深入其数学本质，我们可以观察到两者在“策略更新”这一核心环节上呈现出惊人的同构性。

### 1.1 Multiplicative Weights Update (MWU)

MWU 是在线学习中解决预测专家问题（Prediction with Expert Advice）的经典算法。在 $t$ 时刻，针对各个专家 $i$ 的真实收益 $r_t(i)$，其资金（概率）分配权重更新公式为：


$$w_{t+1}(i) = \frac{w_t(i) \exp(\eta \cdot r_t(i))}{\sum_{j=1}^N w_t(j) \exp(\eta \cdot r_t(j))}$$


其中 $\eta$ 为学习率，控制对近期收益的敏感度。

### 1.2 强化学习中的软策略迭代 (Soft Policy Iteration)

在现代最大熵强化学习（如 SAC）中，为了防止策略过度确定化（Over-determinism）导致的脆弱性，策略 $\pi$ 的提升不再采用贪心算子（Hard-max），而是引入了温度参数 $\alpha$：


$$\pi_{k+1}(a|s) = \frac{\pi_k(a|s) \exp\left( \frac{1}{\alpha} Q^{\pi_k}(s, a) \right)}{Z(s)}$$

### 1.3 数学本质的统一

剥去符号的外衣，这两个公式本质上都是**在线镜像下降（Online Mirror Descent, OMD）**在概率单纯形（Probability Simplex）上的解析解。它们共同优化的是一个带有 **Kullback-Leibler (KL) 散度正则化**的目标函数：


$$\pi_{new} = \arg\max_{\pi \in \Delta} \left( \mathbb{E}_{\pi}[\text{Return}] - \beta \cdot D_{KL}(\pi \parallel \pi_{old}) \right)$$


**核心哲学：** 在追求最大化已知收益的同时，利用 KL 散度作为“引力”，限制策略单次更新的步长，防止在非平稳环境中的剧烈震荡。

## 2. 架构融合：为何互补？如何联合？

既然数学结构一致，为何不合并使用？因为 RL 和 MWU 处理**系统不确定性**的维度是正交的。

### 2.1 互补原理：先验结构 vs 后验对抗

* **RL 的局限与优势：** RL 假设环境是一个平稳的马尔可夫决策过程（MDP）。它的强项在于**状态条件化（State-Conditioned）**，即通过神经网络（Critic）发现“某种宏观状态 $s_t$ 下，某类策略往往有效”的长期结构规律。但这是一种**先验（Prior）**，一旦市场发生规律突变（Regime Shift），RL 会因为对历史过拟合而崩溃。
* **MWU 的局限与优势：** MWU 不假设环境平稳，甚至假定环境是由恶意对手（Adversary）控制的。它的强项在于计算**真实的、无需假设的遗憾最小化（Regret Bounds）**。但它本身是无状态的（Stateless），犹如盲人摸象。它提供的是极其坚固的**后验修正（Posterior Correction）**。

### 2.2 联合公式：元控制器 (Meta-Controller) 的核心逻辑

为了兼顾两者的优势，我们将 RL 的 Critic 网络输出作为基准期望，将 MWU 的累积收益作为对抗性修正，构建出**联合动作下达公式**。

设我们在 $t$ 时刻面临连续可测的市场特征状态 $s_t$，底层有 $N$ 个量化子策略（Alpha 专家）。系统明天分配给专家 $i$ 的资金权重 $w_{t+1}(i)$ 计算如下：


$$w_{t+1}(i) = \frac{\exp\left( \frac{1}{\alpha} Q_{\phi}(s_t, i) + \eta \sum_{\tau=1}^t \tilde{r}_{\tau}(i) \right)}{Z_t}$$

* $Q_{\phi}(s_t, i)$：RL 神经网络基于状态 $s_t$ 输出的**理论期望价值**。
* $\sum \tilde{r}_{\tau}(i)$：MWU 记录的该专家在历史对抗市场中的**真实累积净收益**。
* $\alpha, \eta$：分别控制先验信仰和后验经验的超参数。

**系统运行逻辑：** 当市场平稳，RL 的 $Q$ 值精准，系统顺势攫取超额收益；当黑天鹅降临，RL 给出错误预测，但该专家今天遭受的巨大真实亏损 $\tilde{r}$ 会立刻拉低指数项，MWU 强行压降其权重，守住生存底线。

## 3. 工程实践：硅基落地中的“阿喀琉斯之踵”

将上述极其优美的数学公式写入代码回测时（尤其是在 A 股这种高摩擦、长序列的市场），必须跨越以下几道工程天堑。

### 3.1 致命的数值溢出与 Log-Sum-Exp 技巧

随着时间推移，公式中的指数项 $V_t(i) = \frac{1}{\alpha} Q_{\phi}(s_t, i) + \eta \sum \tilde{r}_{\tau}(i)$ 的绝对值会变得极大或极小。直接计算 $\exp(V_t(i))$ 会导致 `double` 类型溢出，产生 `INF` 或 `NaN`，使系统瞬间崩溃。

**解决方案：** 必须在底层计算算子中强制引入 Log-Sum-Exp (LSE) 技巧。

1. 找出当前项的最大值：$M_t = \max_{j} V_t(j)$
2. 分子分母同时平移：

$$w_{t+1}(i) = \frac{\exp(V_t(i) - M_t)}{\sum_{j=1}^N \exp(V_t(j) - M_t)}$$



经过平移，所有传入 $\exp()$ 的参数都 $\le 0$，彻底杜绝了上溢出的可能，保证了即使运行十年的 tick 级回测，配分函数依然稳定。

### 3.2 真实摩擦与惰性更新 (Lazy Update)

在理论模型中，动作空间是连续单纯形，权重可以无损耗地瞬间调整。但在实战中，每一刻的调仓都伴随印花税（千分之零点五或单边）和滑点。
如果不加限制，MWU 会疯狂调仓，导致账户流血致死。

**解决方案：** 联合公式中的 $\tilde{r}_{\tau}(i)$ 绝对不能是专家的“毛收益”，必须是扣除 $L_1$ 范数位移惩罚后的**严谨净现金流**：


$$\tilde{r}_t = \text{GrossReturn}_t - c \sum_{i=1}^N |w_{t, i} - w_{t-1, i}'|$$


这种基于摩擦成本的 Reward 定义，会迫使 RL 算法学会“延迟满足”，仅当预期收益差级足够大时才触发权重切换。

### 3.3 性能瓶颈与向量化加速

该联合更新模块是高频调用的热点代码。如果状态维度很高且因子数量多，Python 的全局解释器锁（GIL）和循环开销会成为极大瓶颈。
**解决方案：** 在实际部署时，这套元控制器（特别是 LSE 和权重计算模块）强烈建议使用现代 C++（结合 Eigen 库）进行底层的矩阵/向量化重写，通过内存对齐和 SIMD 指令来榨干 CPU 性能。
